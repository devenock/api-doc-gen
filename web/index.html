Generate Swagger/OpenAPI or a Postman Collection from your Go API. No annotations. No code changes. One command.

Get started View on GitHub
~/my-go-api

How it works

From zero to documented API in three steps

01 — Install
One command, any machine

Install globally. Works from any Go project root — no per-project setup required.

go install github.com/devenock/api-doc-gen@latest
02 — Generate
Point and run

Run from your project root. The tool reads your existing routes and handler comments.

api-doc-gen generate --type swagger
03 — Open
Browse or import

Open the Swagger UI in any browser, or drag the Postman collection straight into Postman.

open ./docs/index.html

Output

Two formats. Zero boilerplate.

Swagger / OpenAPI 3.0
Interactive API Explorer

A self-contained Swagger UI you can open in any browser, serve locally, or deploy alongside your service. Machine-readable JSON and YAML included.

openapi.json openapi.yaml index.html — Swagger UI
Postman Collection v2.1
Ready to Test

Endpoints organised into folders by tag. Example request bodies pre-filled. Bearer auth wired up. Import manually or auto-upload via the Postman API.

collection.json folders per route group bearer auth + request examples

Framework support
