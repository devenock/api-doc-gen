api-doc-gen

Scan your Go API and generate Swagger/OpenAPI or a Postman Collection — no annotations, no code changes. Supports Gin, Echo, Fiber, Gorilla Mux, and Chi.

$ go install github.com/devenock/api-doc-gen@latest

Install

Requires Go 1.24+. The binary is installed to $(go env GOPATH)/bin — usually ~/go/bin. Make sure that directory is on your PATH.

# Add to ~/.zshrc or ~/.bashrc if needed
export PATH="$PATH:$(go env GOPATH)/bin"

Verify:

api-doc-gen --version

Generate docs

Run from your Go project root — the directory that contains go.mod and your route definitions.

# Interactive wizard — choose type, output dir, title, etc.
api-doc-gen generate

# Or non-interactive — skip all prompts
api-doc-gen generate --no-interactive --type swagger -o ./docs
api-doc-gen generate --no-interactive --type postman -o ./docs

Point at a project in a different directory:

api-doc-gen generate /path/to/your-api --no-interactive --type swagger -o /path/to/your-api/docs
Run api-doc-gen init inside your project to generate a .apidoc-gen.yaml config file. Commit it so the whole team shares the same defaults.

Optional Makefile target:

docs:
	api-doc-gen generate --no-interactive --type swagger -o ./docs

Output

TypeFiles
swagger openapi.json · openapi.yaml · index.html (Swagger UI)
postman collection.json (Postman Collection v2.1)

Swagger — open docs/index.html directly in your browser, or run api-doc-gen serve ./docs to serve it locally and have it open automatically.

Postman — open Postman, click Import in the sidebar, and drag collection.json onto the dialog. Endpoints are organized into folders by tag, with example request bodies and bearer auth pre-wired.

Flags

FlagDescription
-t, --typeOutput type: swagger or postman
-o, --outputOutput directory (default: ./docs)
-f, --frameworkForce framework: gin · echo · fiber · gorilla · chi
-y, --no-interactiveDisable all prompts — required in CI
--dry-runShow what would be generated, write nothing
--show-configPrint effective config and exit
--uploadUpload Postman collection via API (see Postman)
--titleAPI title (default: auto-detected from your go.mod module name)
--versionAPI version string (default: 1.0.0)
--base-pathPrefix prepended to every path
-v, --verboseShow detected framework, endpoint count, etc.

Full list: api-doc-gen generate --help

Config file

Create .apidoc-gen.yaml in your project root to set persistent defaults. Run api-doc-gen init to generate one interactively.

# .apidoc-gen.yaml
type: swagger
output: ./docs
framework: ""       # leave blank to auto-detect from go.mod
title: My API
version: 1.0.0

Precedence (highest wins): flags → env vars → config file.

Config keyEnv var
typeAPIDOC_TYPE
outputAPIDOC_OUTPUT
frameworkAPIDOC_FRAMEWORK
titleAPIDOC_TITLE
versionAPIDOC_VERSION

Inspect what's loaded: api-doc-gen generate --show-config

Postman

Manual import (default) — generate collection.json, then in Postman click Import in the sidebar and drag the file onto the dialog.

Auto-upload — pass --upload to have the CLI upload the collection directly to your Postman workspace via the Postman API. You'll be prompted for an API key the first time; generate one at postman.co/settings/me/api-keys. The key is saved to ~/.config/apidoc-gen/credentials.json and reused on future runs. Repeat runs update the same collection — no duplicates.

# First run — prompts for key once, then uploads
api-doc-gen generate --no-interactive --type postman --upload -o ./docs

# CI — read key from env, fail loudly if missing
APIDOC_POSTMAN_API_KEY=your_key api-doc-gen generate -y --type postman --upload

# Generate file only, skip upload entirely
api-doc-gen generate -y --type postman --no-upload

The collection UID is cached in .apidoc-gen-cache.json (gitignored) so subsequent uploads always update the same collection.

Docker

Run without installing Go locally. Mount your project as the workspace:

docker build -t apidoc-gen .

docker run --rm \
  -v "$(pwd)":/workspace \
  -w /workspace \
  apidoc-gen generate --no-interactive --type swagger -o ./docs

Replace $(pwd) with the path to your API project if you're not running from its root.

CI & scripts

Always pass -y and --type so the CLI never waits for input.

# GitHub Actions example
- name: Generate API docs
  run: |
    go install github.com/devenock/api-doc-gen@latest
    api-doc-gen generate -y --type swagger -o ./docs --title "My API"

Exit codes: 0 success · 1 validation error · 2 runtime error.

Troubleshooting

Command not found after install
Go's bin directory is not on your PATH. Add it and reload your shell:

export PATH="$PATH:$(go env GOPATH)/bin"
source ~/.zshrc   # or ~/.bashrc

No endpoints found
Pass --framework explicitly (e.g. --framework gin), or run with -v to see what was detected. Confirm route files are not inside an excluded directory (vendor, test).

Config not picked up
Ensure .apidoc-gen.yaml is in the directory you're running from, not a subdirectory. Inspect with api-doc-gen generate --show-config.

Prompts appearing in CI
Always pass -y (--no-interactive) and --type. Without these the CLI waits for interactive input.
